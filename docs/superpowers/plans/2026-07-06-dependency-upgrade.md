# Dependency Upgrade Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Bring every dependency in `id-game` up to today's latest stable versions (2026-07-06), starting by completing the in-flight Next.js 15→16 and Zod 3→4 upgrades, so the app builds cleanly and Dependabot goes quiet.

**Architecture:** Work on a fresh branch cut from `main` (the stale `deps/update-30-dec-2025` branch is abandoned — its learnings are folded in here, then it and the other dead branches are deleted). The upgrade is delivered in three independently-mergeable phases in ascending risk order: **Phase 1** completes Next 16 + Zod 4 (the priority, self-contained); **Phase 2** takes all remaining low/medium-risk deps to latest (minors, lucide 1.x, ESLint 10, TypeScript 6); **Phase 3** is the high-effort, decision-gated Clerk 6→7 (Core 3) migration, which requires rebuilding the auth UI off the deprecated `@clerk/elements`.

**Tech Stack:** Next.js (App Router) · React 19 · Convex · Clerk · PostHog · Tailwind v4 · shadcn/ui · react-hook-form + Zod · TypeScript · pnpm.

## Global Constraints

- **Package manager:** `pnpm@9.15.4` (already pinned in `package.json > packageManager`). Use `pnpm`, never `npm`/`yarn`, for installs.
- **Node floor:** `>=20.19` (ESLint 10 is the strictest consumer; Next 16 and Clerk 7 want ≥20.9). Installed on this machine: v25.8.1 — satisfies it.
- **Target versions (npm `latest`, verified 2026-07-06):** next `16.2.10` · react/react-dom `19.2.7` · zod `4.4.3` · @hookform/resolvers `5.4.0` · react-hook-form `7.81.0` · @clerk/nextjs `6.39.5` (Phase 1/2) then `7.5.12` (Phase 3) · convex `1.42.1` · @convex-dev/migrations `0.3.5` · eslint `10.6.0` · eslint-config-next `16.2.10` · typescript `6.0.3` · lucide-react `1.23.0` · @types/node `^25`.

Fix with the documented three-generic `z.input`/`z.output` form. `join-game.tsx` has no `.default()` and needs no change.
